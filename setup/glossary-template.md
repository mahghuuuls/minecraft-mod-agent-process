# Project Glossary

This document defines project-specific terms and preferred vocabulary used by the mod project.

Keep the active glossary under:

```text
workspace/documentation/glossary.md
```

The glossary is for terms whose meaning affects requirements, behavior, configuration, architecture, code naming, public documentation, or owner communication. It is not meant to define every general Minecraft, programming, or workflow term.

## Glossary Rules

- Use one preferred term for one concept.
- Clarify cases where two terms appear to mean the same thing.
- Clarify cases where one term appears to have multiple meanings.
- Add candidate terms when project-specific vocabulary appears.
- Approve or revise terms when they affect requirements, architecture, code naming, configuration, or release copy.
- Mark outdated terms instead of deleting them when older documents, code, or public copy may still mention them.
- Avoid adding obvious general terms unless the project uses them in a special way.
